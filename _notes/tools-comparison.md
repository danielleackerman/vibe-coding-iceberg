## üîß Vibe Coding Tools (AI‚Äëpowered IDEs)

### **Replit**
- **Target users**: Beginners, educators, small teams  
- **What it does**: Browser-based IDE with AI agents that scaffold full-stack apps from prompts  
- **Strengths**: One-shot prototypes, built-in hosting & database  
- **Limitations**: Not ideal for custom or complex production systems

### **Cursor**
- **Target users**: Experienced developers, pro teams  
- **What it does**: AI-enhanced local IDE built on VS Code  
- **Strengths**: Deep integration, powerful AI features, multi-LLM support  
- **Limitations**: Steep learning curve, more technical setup

### **Bolt**
- **Target users**: Mid-level product builders  
- **What it does**: Full-stack browser-based builder for UI and app scaffolding  
- **Strengths**: Figma-faithful UI builds, easy Next.js generation  
- **Limitations**: Occasional instability, less flexible than Cursor

### **Lovable.dev**
- **Target users**: Non-technical founders  
- **What it does**: Prompt-to-product pipeline using Supabase and plug-and-play backend  
- **Strengths**: MVP creation without code  
- **Limitations**: Some black-box behavior

### **v0 by Vercel**
- **Target users**: Frontend devs and designers  
- **What it does**: Prompts to clean React + Tailwind UI components  
- **Strengths**: Clean output, works well with design tools  
- **Limitations**: Less customizable, fixed styling logic

## üß© Emerging Vibe Design Tools (Prompt-to-UI)

### **Subframe**
- **Use cases**: UI designers, dev/design hybrids  
- **What it does**: Drag-drop editor ‚Üí exports real JSX/Tailwind/Next.js  
- **Strengths**: Visual-first but outputs real code  
- **Limitations**: Early-stage, evolving toolset

### **Aura**
- **Use cases**: Designers creating Figma/HTML quickly  
- **What it does**: Prompts generate responsive UIs and animations  
- **Strengths**: Great for mockups, quick ideas  
- **Limitations**: Needs post-processing for full systems

### **MagicPath**
- **Use cases**: AI-assisted UI layouts  
- **What it does**: Suggests layout and structure using content and heuristics  
- **Strengths**: Early taste of layout intelligence  
- **Limitations**: Very new; capabilities still evolving

## üîç Key Differences

### 1. Coders vs Designers
- Vibe coding = code-first (Replit, Cursor)
- Vibe design = design-first (Subframe, Aura)

### 2. Level of Control
- **Cursor**: Deep control
- **Replit/Lovable**: Managed setup
- **Bolt/v0**: Middle ground
- **Subframe/Aura**: Visual-first, code export optional

### 3. Output Type
- **Vibe coding**: Full-stack logic + hosting
- **Vibe design**: UI/UX mockups + component code

### 4. Deployment & Hosting
- **Code platforms**: Often auto-deployed (Replit, Lovable, Bolt)
- **Design platforms**: Exported code, requires your own hosting
